# PRODUCTION_NOTES.md

## 适用场景
当前版本适合：
- 内网服务
- 小规模团队内部使用
- 单机容器部署
- 规则可控、文件类型明确的场景

不建议直接裸上公网。

## 上线前最低检查项
1. 设置 `API_KEY`
2. 确认 `config/rules.yaml` 与词典内容
3. 执行 `pytest`
4. 执行一次真实上传验证
5. 检查 `logs/` 是否正常输出
6. 检查 `outputs/` 是否能正常生成文件、报告、zip

## 风险点
### 1. 样式保留风险
- docx 为解决跨 run 命中，采用段落合并再回写
- 这会影响复杂 run 级样式

### 2. 文件类型边界
- 不支持 pdf、图片、扫描件
- 上传这些内容时应在前端或调用侧提前拦截

### 3. 安全边界
当前已具备：
- 后缀白名单
- 文件大小限制
- 签名校验（docx/xlsx）
- API Key 鉴权
- UUID 落盘

当前仍缺：
- 限流
- 更完整的 MIME 嗅探
- 审计告警
- 访问日志分析
